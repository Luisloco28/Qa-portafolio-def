## 🧪 Sobre este portafolio

Este repositorio reúne proyectos prácticos donde aplico el ciclo completo de pruebas de software: análisis de requisitos, diseño de casos de prueba, ejecución y reporte de errores.

Cada proyecto incluye:
- Análisis de requisitos
- Diseño de casos de prueba (positivos, negativos, valores límite y clases de equivalencia)
- Ejecución de pruebas (manuales y vía API)
- Reportes de bugs documentados con pasos de reproducción, resultado actual y esperado

---

## 📂 Proyectos

### 🖥️ [Pruebas de UI — Urban Routes](./sprint-3-ui-testing)
Pruebas manuales sobre la interfaz de una app de transporte (Urban Routes). Incluye checklist de diseño, casos de prueba de lógica de negocio y 17 reportes de bugs documentados en Jira.

**Habilidades demostradas:** pruebas exploratorias, checklist de UI, valores límite, redacción de bug reports reproducibles.

---

### 🔌 [Pruebas de API — Urban Grocers](./sprint-4-api-testing)
Pruebas funcionales de una API REST (kits y servicios de entrega) ejecutadas con Postman. Incluye 34 casos de prueba y 13 bugs encontrados relacionados con validación de datos y reglas de negocio.

**Habilidades demostradas:** pruebas de API con Postman, validación de status codes, clases de equivalencia, valores límite, manejo de errores 400/404/500.
